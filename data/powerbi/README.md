# GridSentinel — Power BI dataset

Star-schema export plus two Power BI deliverables: a `.pbit` template
(best-effort, opens directly in Power BI Desktop) and a `.pq` Power Query
script (guaranteed-to-paste fallback if the template won't open).

## What's in this folder

| File | Purpose |
|---|---|
| `GridSentinel.pbit` | **Power BI template** (Windows-only). Pre-defines all 6 tables, relationships, and 10 DAX measures (Alert Compression, MTTR, Perf/Watt, Idle Waste USD, etc). Generated by `gridsentinel.bi.pbit`. |
| `GridSentinel.pq` | **Power Query M script** — guaranteed-working alternative. Paste into Power BI's Advanced Editor; edit the `DataFolder` line; done. |
| `measures.dax` | Reference list of DAX measures (the .pbit embeds these already; this file is the human-readable version). |
| `*.parquet`, `*.csv` | Star-schema flat files. Use parquet for performance, CSV for tools that don't speak parquet. |

## How to use

### Path 1 — open the .pbit (Windows, Power BI Desktop)

```
Open Power BI Desktop → File → Open → GridSentinel.pbit
```

The template loads all 6 tables, relationships, and measures. The
DataMashup queries reference `./data/powerbi/*.csv` relative to where you
opened the file — adjust the path inline if Power BI prompts you to refresh
data sources.

### Path 2 — paste the .pq script (the reliable fallback)

If the .pbit refuses to open (Power BI Desktop is finicky about template
format), use this instead:

1. Power BI Desktop → `Home` → `Transform data` → `New Source` → `Blank Query`
2. `Home` → `Advanced Editor`
3. Paste the entire contents of `GridSentinel.pq`
4. Change the `DataFolder` line at the top to your absolute path
5. `Done`

All 6 tables will appear in the queries pane. Then add the measures from
`measures.dax` via `Modeling → New Measure`.

## macOS users

Power BI Desktop is Windows-only. Three workarounds:

- **Power BI Service** (browser) — requires a Microsoft work/school account
  (consumer Gmail won't sign up). Upload CSVs there and build visuals in
  the web editor.
- **Parallels / UTM** — run a Windows VM, install Power BI Desktop free.
- **Skip Power BI** and use the [Tableau Public workbook](../tableau/GridSentinel.twbx) instead — that one opens on macOS natively.

## Regenerating

```bash
python -m gridsentinel.powerbi.export   # regenerate parquet/CSV from sim
python -m gridsentinel.bi.pbit          # rebuild GridSentinel.pbit + .pq
```
